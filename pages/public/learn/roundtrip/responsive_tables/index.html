    responsive tables

    Creating a design for responsive tables is challenging. The approch used by J1 Template is based on pure CSS on top of the classic Bootstrap styles for simplicity and portability to be viewed best on all devices and browsers.

    Table types

    But, for sure, the solution provided is a compromise. To fit the common needs for tables viewed on desktop and mobile devices, three types of responsive tables are implemented:

    Table 1. Types of responsive tables
    Type Style Description

    Bootstrap

    table-responsive

    The adding the class table-responsive, it creates a responsive table based on BS CSS styles. The table will then scroll horizontally on small devices. Making any table responsive across all viewports, the additonal class table-responsive is to applied. For specific (Bootstrap) breakpoints, the classes table-responsive-lg{-sm|-md|-lg|-xl} are available to have better control on what viewport sizes tables are being transformed.

    R Tables

    r-text-xxx

    R tables are used by J1 Template for default. Those tables are using the responsive text feature (presented in section Typography) with no additonal CSS classes to be applied. For smaller viewports, the text scales down and should fit the cells automatically. The default text size for J1 is r-text-300.

    F Tables

    no-r-text

    If the responsive text feature from J1 is not wanted, the CSS class no-r-text is to be applied. F (fixed) tables are not responsive per default, but in combination with the CSS class table-responsive-stacked-lg (Bootstrap), this type behaves like classic Bootstrap responsive table (see above).

    S Tables

    table-responsive-stacked-lg-stacked

    The style table-responsive-stacked-lg-stacked rebuilds a table. The header is set to invisible and all columns gets vertically stacked as rows. For specific (Bootstrap) breakpoints, the classes table-responsive-stacked-lg-stacked{-sm|-md|-lg|-xl} are available to have better control on what viewport sizes tables are being transformed. The class table-responsive-stacked-lg-stacked is using the breakpoint at 768px what is the same as table-responsive-stacked-lg-stacked-md

    The styles for responsive tables takes effect for default if the viewport (window size) is smaller than 768px for the width (x-axis). Typical for small devices like tablets or mobiles.

    The styles for F and S Tables can be combined with the Bootstrap responsive table approach (table-responsive-stacked-lg).

    To see the transformation o table in action, resize for desktop devices the browser window or use a device the viewport is smaller than 768px (mobiles or tablets).

    Multi-column Table

    Responsive Bootstrap tables support tables to be scrolled horizontally on smaller viewports. Making any table responsive across all viewports, the additonal class table-responsive-stacked-lg is used. For specific (Bootstrap) breakpoints, the classes table-responsive-stacked-lg{-sm|-md|-lg|-xl} are available to have better control on what viewport sizes table are transformed. From a specified breakpoint and up, the table will behave normally and do not scroll horizontally.

    Bootstrap responsive tables make use of overflow-y: hidden, which clips off any content that goes beyond the bottom or top edges of the table. In particular, this can clip off dropdown menus and other third-party widgets.
